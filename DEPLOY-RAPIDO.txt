DEPLOY RÁPIDO — SALVATORE

1. Crie um repositório GitHub e envie esta pasta.
2. Cloudflare: crie um banco D1 chamado salvatore-compras.
3. Cloudflare Pages: conecte o GitHub.
   Build command: vazio
   Output directory: public
4. No projeto Pages, crie o binding D1:
   variável DB -> banco salvatore-compras
5. Crie apenas o secret:
   ADMIN_TOKEN
6. Refaça o deploy.
7. App: endereço principal
8. Administração: /admin.html

O aplicativo principal é público e não pede PIN.
Somente /admin.html exige ADMIN_TOKEN.

Os dados ficam no D1 e continuam salvos mesmo após novos deploys.


ATUALIZAÇÃO DA ABA VINHOS
- Substitua os arquivos no GitHub e faça commit.
- O Cloudflare fará o deploy automaticamente.
- Abra o site uma vez para executar a migração automática.
- Depois use /admin.html e escolha o setor Vinhos.


ATUALIZAÇÃO DE ORGANIZAÇÃO
1. Substitua os arquivos do GitHub por esta versão.
2. Faça commit na branch main.
3. Aguarde o deploy automático.
4. Abra a página principal uma vez.
5. A migração do D1 será executada automaticamente.
6. Não apague o banco e não execute SQL manualmente.


ATUALIZAÇÃO DAS SUBABAS
- Substitua os arquivos no GitHub e faça commit.
- Não precisa alterar o banco D1.
- As subabas são geradas automaticamente pelas categorias existentes.


ATUALIZAÇÃO DE CUSTOS E RESPONSIVIDADE
1. Substitua os arquivos no GitHub e faça commit.
2. Não altere nem apague o banco D1.
3. Abra o site após o deploy; a coluna de custo será criada automaticamente.
4. Entre em /admin.html e edite os itens para informar o custo unitário.


CORREÇÃO DO CUSTO
- Substitua os arquivos no GitHub.
- Faça commit e aguarde o deploy.
- Abra /admin.html, edite um item e informe o custo.
- O painel deve confirmar o custo salvo no D1.


PREÇO V3 — IMPORTANTE
1. Substitua no GitHub a pasta public.
2. Substitua também a pasta functions.
3. Faça commit na main.
4. Aguarde o deploy ficar Success.
5. Faça Retry deployment se necessário.
6. Abra /admin.html em aba anônima.
7. A página deve mostrar “versão preço v3”.
8. Edite um custo e confira a mensagem “confirmado no D1”.


CORREÇÃO PREÇO V4
1. Substitua public e functions no GitHub.
2. Faça commit na main.
3. Aguarde o deploy Success.
4. Abra /admin.html em janela anônima.
5. Confirme o texto “versão preço v4”.
6. Salve um custo e confira “salvo e confirmado no D1”.


PERSISTÊNCIA V5
1. Substitua public e functions no GitHub.
2. Faça commit na main.
3. Na Cloudflare, confirme o binding Production:
   Variable name: DB
   Database: salvatore-compras
4. Faça novo deploy depois de conferir o binding.
5. Abra somente:
   https://lista-compras-aiq.pages.dev/admin.html
6. O painel mostrará o domínio e o identificador D1.
